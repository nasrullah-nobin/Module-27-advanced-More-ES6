## 27-2 Different truthy and falsy values in JavaScript

১. if (value) দিয়ে চেক করলে জাভাস্ক্রিপ্ট ভ্যালুটা truthy না falsy সেটা দেখে সিদ্ধান্ত নেয়।

২. Falsy ভ্যালুগুলো হলো:

false

0 এবং -0

"" (empty string)

null

undefined

NaN

৩. এগুলো ছাড়া বাকি প্রায় সব ভ্যালুই truthy। যেমন:

"0" (স্ট্রিং)

"false" (স্ট্রিং)

" " (স্পেসসহ স্ট্রিং)

{} (empty object)

[] (empty array)

ফাংশন

৪. খালি স্ট্রিং "" falsy, কিন্তু স্পেসসহ " " truthy।

৫. !value দিলে ভ্যালুর বিপরীত বুলিয়ান পাওয়া যায় (true → false, false → true)।

৬. !!value দিলে যেকোনো ভ্যালুকে বুলিয়ান (true/false) এ কনভার্ট করা যায়।

মোট কথা: ৬টা নির্দিষ্ট falsy ভ্যালু মনে রাখলেই বাকি সব truthy ধরা যায়।

## 27-3 Double equal ( == ) vs triple equal ( === ), implicit conversion

== vs === — Short Summary

১. === (Triple Equal) ব্যবহার করাই সেফ এবং রিকমেন্ডেড।

ভ্যালু এবং টাইপ দুইটাই চেক করে

টাইপ আলাদা হলে সরাসরি false

কোনো টাইপ কনভারশন করে না
একে বলে Strict Equality

২. == (Double Equal) ঝামেলার জায়গা।

আগে টাইপ কনভার্ট করে, তারপর ভ্যালু চেক করে

একে বলে Type Coercion
একে বলে Loose Equality

Double Equal (==) এর ঝামেলা

2 == "2" → true

1 == true → true

0 == false → true

null == undefined → true

NaN == NaN → false

{ } == { } → false (কারণ আলাদা রেফারেন্স)

কারণ: == ডিফারেন্ট টাইপ পেলে আগে কনভার্ট করে নেয়, তারপর তুলনা করে।

## 27-4 Block scope, global scope, simple understanding of hoisting

১. Scope মানে হলো কোন ভেরিয়েবল বা ফাংশন কোথা পর্যন্ত ব্যবহার করা যাবে তার সীমানা।

২. ফাংশনের ভিতরে ভেরিয়েবল ডিক্লেয়ার করলে সেটা শুধু সেই ফাংশনের ভিতরেই ব্যবহার করা যায় → এটাকে Function/Local Scope বলে।

৩. ফাংশনের বাইরে ভেরিয়েবল ডিক্লেয়ার করলে সেটা সব জায়গা থেকে ব্যবহার করা যায় → এটাকে Global Scope বলে।

৪. ফাংশনের ভিতরের ভেরিয়েবল বাইরে থেকে অ্যাক্সেস করলে Reference Error হবে।

৫. বাইরের (Global) ভেরিয়েবল ফাংশনের ভিতর থেকে ব্যবহার করা যায়।

৬. ভেরিয়েবল খুঁজতে গেলে জাভাস্ক্রিপ্ট আগে নিজের স্কোপে খোঁজে, না পেলে উপরের স্কোপে খোঁজে → এটাকে Scope Chain বলে।

৭. { } ব্র্যাকেট দিয়ে তৈরি কোড ব্লক (যেমন if, for) নিজের আলাদা স্কোপ তৈরি করে → এটাকে Block Scope বলে।

৮. let এবং const দিয়ে ডিক্লেয়ার করা ভেরিয়েবল ডিক্লেয়ার করার আগে ব্যবহার করলে এরর দেয় → এটাকে Temporal Dead Zone (TDZ) বলে।

৯. ফাংশন ডিক্লেয়ার করলে সেটা আগে থেকেই ব্যবহার করা যায় → এটাকে Hoisting বলে।

১০. let এবং const হোইস্ট হলেও ডিক্লেয়ার করার আগে ব্যবহার করা যায় না (TDZ এর কারণে)।

মূল কথা

ভিতরের জিনিস বাইরে ব্যবহার করা যায় না।

বাইরের জিনিস ভিতরে ব্যবহার করা যায়।




## 27-5 (Advanced) Explore closure in details

১. ক্লোজার হলো এমন একটি ফিচার যেখানে একটি ফাংশন তার আউটার ফাংশনের ভেরিয়েবল মনে রাখতে পারে, এমনকি আউটার ফাংশন শেষ হয়ে গেলেও।

২. যখন কোনো ফাংশনের ভিতরে আরেকটি ফাংশন ডিক্লেয়ার করে সেটাকে রিটার্ন করা হয়, তখন ইনার ফাংশন তার আউটার স্কোপের ভেরিয়েবল ধরে রাখে।

৩. আউটার ফাংশন কল করলে একটি নতুন আলাদা এনভায়রনমেন্ট (scope) তৈরি হয়।

৪. রিটার্ন করা ইনার ফাংশন বারবার কল করলে সে আগের ভ্যালু মনে রাখে এবং আপডেট করতে পারে।

৫. প্রতিবার আউটার ফাংশন নতুন করে কল করলে আলাদা আলাদা ক্লোজার তৈরি হয় (যেমন: রহিমের কাউন্টার আর করিমের কাউন্টার আলাদা)।

৬. বাইরে থেকে সরাসরি ভিতরের ভেরিয়েবল অ্যাক্সেস করা যায় না, কিন্তু ইনার ফাংশনের মাধ্যমে পরিবর্তন করা যায়।

৭. ক্লোজার ব্যবহার করা হয় ডাটা প্রাইভেট রাখা, কাউন্টার বানানো, এবং এনক্যাপসুলেশন টাইপ আচরণ তৈরির জন্য।

মূল কথা

ক্লোজার মানে হলো—একটি ফাংশন তার তৈরি হওয়ার সময়কার স্কোপকে “মনে রাখে” এবং পরে সেটি ব্যবহার করতে পারে।
